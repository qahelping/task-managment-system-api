# 📚 Инструкция по развертыванию после фикса багов или обновления проекта

### Запуск приложения через Docker

1. **Перейдите на нужную ветку в проекте:**
   ```bash
   cd task-managment-system-api
   git fetch
   git switch v2_fix_bugs
   ```

2. **Запустите контейнеры:**
   ```bash
   docker-compose up --build
   ```

   Флаг `--build` пересоберет образы при первом запуске или после изменений.

3. **Для запуска в фоновом режиме (detached mode):**
   ```bash
   docker-compose up -d --build
   ```

4. **Проверьте статус контейнеров:**
   ```bash
   docker-compose ps
   ```

✅ После запуска приложение будет доступно:
- **Frontend:** http://localhost:3000
- **Backend API:** http://localhost:8000
- **API Документация:** http://localhost:8000/docs
